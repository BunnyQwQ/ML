# Connect Four — MCTS и AlphaZero

Два подхода к игровому ИИ для Connect Four (4-в-ряд), от классического дерева
поиска к обученному через self-play. Оба написаны с нуля и умеют играть против
человека в консоли.

- [`MCTS.cpp`](MCTS.cpp) — **классический Monte Carlo Tree Search** (C++, без нейросети)
- [`AlphaZero.py`](AlphaZero.py) — **AlphaZero**: MCTS под управлением обученной сети (PyTorch)
- `alphazero_c4.pt` — обученные веса AlphaZero (готовы к игре)

---

## 1. MCTS (`MCTS.cpp`)

Классический MCTS с четырьмя фазами на каждой симуляции:

1. **Selection** — спуск по дереву по формуле UCT (`-Q + c·√(ln N / n)`)
2. **Expansion** — добавление одного случайного неиспробованного хода
3. **Simulation** — случайный доигрыш (rollout) до конца партии
4. **Backpropagation** — протаскивание результата вверх со сменой знака у соперника

Оценка позиции берётся из **случайных доигрышей** — никакого обучения. Сила — от
числа симуляций и скорости.

- **Битборды** (`uint64_t` на игрока) — компактное представление, проверка победы
  через битовые сдвиги (очень быстро)
- Пул узлов на `std::vector` (индексы вместо указателей) — дружелюбно к кэшу
- Интерактивная игра, настраиваемое число симуляций (по умолчанию 20000), вывод
  посещений/винрейта/скорости (сим/с)

## 2. AlphaZero (`AlphaZero.py`)

MCTS, в котором случайные роллауты заменены **обученной нейросетью**, а сама сеть
учится на партиях, которые агент играет сам с собой (self-play).

**Сеть** (`AlphaZeroNet`) — ResNet-стиль:
- Тело: conv-stem + 4 остаточных блока (`ResBlock`)
- **Policy-голова** → приоры ходов `π(a|s)`
- **Value-голова** → оценка позиции `v ∈ [−1, 1]` (tanh)

**Поиск** — PUCT: `Q + c_puct · P · √N / (1 + n)`, где приоры `P` даёт policy-голова,
а ценность листа — value-голова (доигрыш не нужен).

**Обучение (self-play):**
- Партии с собой; **шум Дирихле** в корне для исследования
- **Температурный** выбор хода на первых `temp_moves` ходах (разнообразие), дальше `argmax`
- Replay-буфер; loss = MSE(value) + кросс-энтропия(policy) против визитов MCTS
- Целевое значение `z` — исход партии, чередующий знак по ходам

## Сравнение

| | MCTS (`.cpp`) | AlphaZero (`.py`) |
|---|---------------|-------------------|
| Оценка позиции | случайные роллауты до конца | value-голова сети |
| Выбор хода | UCT | PUCT (приоры от policy-головы) |
| Обучение | нет | self-play + градиентный спуск |
| Представление | битборд (`uint64`) | тензор `2×6×7` |
| Язык | C++ | Python / PyTorch |
| Источник силы | число симуляций + скорость | качество обученной сети |

★ Ключевая идея AlphaZero: обученная value-сеть заменяет тысячи случайных
доигрышей одной оценкой, а policy-голова сразу направляет поиск в перспективные
ветки — та же (или выше) сила при **на порядок меньшем** числе симуляций.

## Запуск

**AlphaZero** (Python: `torch`, `numpy`):
```bash
python RL/connect_four/AlphaZero.py play     # игра против обученной модели
python RL/connect_four/AlphaZero.py train    # обучение с нуля (self-play)
```
Полезные флаги: `--simulations`, `--c-puct`, `--iterations`, `--games`, `--resume`.

**MCTS** (C++17, компиляция; код Windows-специфичен — использует `windows.h` для UTF-8 консоли):
```bash
g++ -O2 -std=c++17 RL/connect_four/MCTS.cpp -o mcts
./mcts
```

## Гиперпараметры AlphaZero (по умолчанию)

| Параметр | Значение | Смысл |
|----------|----------|-------|
| `channels` / `blocks` | 64 / 4 | ширина и глубина ResNet |
| `simulations` | 200 | симуляций MCTS на ход |
| `c_puct` | 1.5 | баланс exploration/exploitation в PUCT |
| `games` / `iterations` | 20 / 50 | партий self-play на итерацию / итераций |
| `temp_moves` | 10 | ходов с температурным сэмплированием |
