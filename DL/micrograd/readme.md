# Micrograd — Autograd Engine from Scratch

Реализация небольшого автоград-движка и нейросети поверх него, по мотивам [Karpathy's micrograd](https://github.com/karpathy/micrograd).

## Что реализовано

- **`Value`** — класс-обёртка над скаляром, строящий вычислительный граф "на лету". Поддерживает базовые операции (`+`, `*`, `**`, `tanh` и т.д.) и хранит локальное правило дифференцирования (`_backward`) для каждой операции.
- **`backward()`** — обратное распространение градиента через граф: топологическая сортировка узлов + последовательный вызов `_backward()` в обратном порядке, с аккумуляцией градиентов (`+=`) для узлов с несколькими потребителями.
- **MLP** (`Neuron` → `Layer` → `MLP`) — простая полносвязная сеть, построенная целиком на `Value`, без PyTorch — прямой проход и градиенты весов считаются через тот же движок.

## Проверка на практике

Обучил сеть на **бинарной классификации**: находится ли точка `(x, y)` внутри круга.

- Сгенерирован игрушечный датасет точек с метками (внутри/снаружи круга)
- Ручной цикл обучения: forward → loss → `backward()` → обновление весов → `zero_grad`
- Loss стабильно падает, сеть сходится к разумной границе решения

## Ключевые выводы

- `_backward` каждого узла — это локальный VJP (vector-Jacobian product) для конкретной операции, а не пересчёт значения
- Топосортировка гарантирует, что градиент в узел придёт полностью просуммированным до того, как узел передаст его дальше назад
- Ручная реализация backprop на скалярах напрямую переносится на понимание того, как работает `loss.backward()` в PyTorch
