# mini-GPT — трансформер посимвольной генерации с нуля

Decoder-only трансформер (в стиле GPT), реализованный с нуля на PyTorch и обученный генерировать имена посимвольно. Все ключевые компоненты — self-attention, multi-head attention, трансформер-блок, позиционные эмбеддинги — написаны вручную, без `nn.Transformer`, чтобы разобраться в архитектуре на уровне отдельных операций над тензорами.

Финальный проект в цепочке обучения `micrograd → makemore → mini-GPT`.

## Что реализовано

Модель собрана снизу вверх из компонентов, каждый написан вручную:

- **Single-head self-attention** (`Head`) — scaled dot-product attention с causal-маской. Проекции query/key/value, `softmax(QKᵀ / √d_k)` и нижнетреугольная маска, запрещающая позиции смотреть на будущие токены.
- **Multi-head attention** (`MultiHeadAttention`) — несколько голов внимания параллельно, их выходы конкатенируются и смешиваются выходной проекцией. Размерность эмбеддинга делится между головами (`head_size = C / num_heads`).
- **Feed-forward сеть** (`FeedForward`) — позиционный MLP с расширением ×4, обрабатывает каждый токен отдельно после того, как attention собрал информацию между токенами.
- **Трансформер-блок** (`Block`) — подслои attention и feed-forward, каждый с pre-LayerNorm и residual-связью.
- **Полная модель** (`GPT`) — токенные + позиционные эмбеддинги, стек трансформер-блоков, финальный LayerNorm и линейная голова с проекцией на словарь. Используется weight tying между входным эмбеддингом и выходной головой.

## Результаты

Обучение на датасете из ~5000 имён (посимвольно, русский язык).

| Модель | Test loss (кросс-энтропия) |
|--------|----------------------------|
| MLP (предыдущий проект) | 2.193 |
| **mini-GPT (этот проект)** | **2.019** |

Трансформер улучшает baseline MLP. На датасете такого размера выигрыш скромный — цель проекта в корректной реализации с нуля и понимании архитектуры, а не в максимизации метрики.

### Примеры генерации

```
тамурахф, гаффа, илонпер, сера, серхине,
тамбан, мила, амыкбанир, жаналия, фанде
```

Модель выдаёт правдоподобные имяподобные строки — часть похожа на настоящие имена (сера, мила, жаналия), часть — фонетически согласованные выдуманные варианты.

## Ключевые детали реализации

- **Causal-маскирование** через `masked_fill(tril == 0, -inf)` перед softmax: `e^{-inf} = 0` обнуляет внимание к будущим позициям. Маска — зарегистрированный буфер (не обучаемый параметр), поэтому обучение не может её сломать.
- **Масштабирование attention** на `1/√head_size`, чтобы скалярные произведения не уводили softmax в насыщение.
- **Обучение по всей последовательности** — каждое имя обучается как последовательность с предсказанием следующего токена на *каждой* позиции сразу (loss по всем позициям), а не одно предсказание на фиксированное окно.
- **Обработка паддинга** — имена переменной длины дополняются до `block_size`. Цели паддятся значением `-1` с `ignore_index=-1` в loss, поэтому токен конца имени (`0`) остаётся живой целью обучения, а чистый паддинг игнорируется.
- **Инициализация весов** — `std=0.02` на слоях Linear и Embedding (конвенция GPT-2), что приближает начальный loss к теоретическому `ln(vocab_size)`.
- **Стабильность обучения** — AdamW с weight decay, клиппинг градиентов (`max_norm=1.0`) и early stopping с восстановлением весов лучшей эпохи по валидации.

## Схема архитектуры

```
токены → токенный эмбеддинг + позиционный эмбеддинг
       → [трансформер-блок] × N
            блок: x = x + attention(LayerNorm(x))
                  x = x + feedforward(LayerNorm(x))
       → финальный LayerNorm
       → линейная голова → логиты по словарю
```

## Запуск

```python
from mini_gpt import GPTFinal

m = GPTFinal(file_path="names_table.jsonl")
m.train_model()
m.plot_losses()
print("test loss:", m.test_loss())
print(m.generate())
```

| `batch_size` | 32 |
